# Workflow Comparison

For this assignment, I built the same React settings form using two different prompting approaches.

## Round 1 – Vague Prompt

The first version was created using a very simple prompt asking for a React settings form. The generated result was functional but basic. It contained only the essential UI elements and required additional review to identify missing features. Validation, accessibility, and user feedback were minimal, meaning users could submit incomplete or invalid information. Although the code worked, it required more manual checking and improvements before it could be considered production-ready.

## Round 2 – Detailed Prompt

For the second version, I used a much more detailed approach by specifying the expected behavior, validation requirements, and overall functionality. The improved version included form validation, email validation, error messages, a theme selector, notification preferences, and better styling. The structure was cleaner, making the code easier to understand and maintain.

## Comparison

The second approach produced more complete and reliable code. It reduced the amount of manual review required because many edge cases were already handled. Validation prevented invalid input, while clearer structure improved readability. The user interface was also more polished and accessible.

One issue I encountered during the assignment was Git branch switching on Windows due to file-locking and OneDrive synchronization. I resolved the issue by restoring the working tree, ensuring no development servers were running, and completing the remaining work on separate branches.

Overall, this exercise showed that investing time in writing a precise prompt produces higher-quality code and reduces the overall development effort.
